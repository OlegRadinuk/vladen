## Ключевые принципы
- Mobile first: 375px → 768px → 1280px+
- Токены и ключи только в `.env.local`, не коммитить
- SEO: метатеги + openGraph на каждой странице, sitemap.xml, robots.txt
- Форма отправляет на `/api/telegram` (имя + телефон, проверка origin, только POST)
- LoadingContext (`contexts/LoadingContext.tsx`) — CountUp ждёт пока лоадер не уйдёт

## АДАПТИВНОСТЬ — КРАСНАЯ ЛИНИЯ
Каждый новый компонент обязан проверяться на 375px, 640px, 1280px.
- Заголовки ВСЕГДА через три ступени: `text-3xl sm:text-4xl md:text-5xl` — пропускать sm нельзя
- `flex-shrink-0` на фиксированных элементах хедера (лого, бургер), иначе прыгает на узких экранах
- Паддинги карточек: `p-6 sm:p-8` — не `p-8` без мобильного варианта
- `gap` в flex/grid на мобиле уменьшать: `gap-6 sm:gap-10`
- Текст внутри хедера скрывать на xs если занимает место: `hidden sm:block`
